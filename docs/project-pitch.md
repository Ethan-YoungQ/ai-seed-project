## 一、培训行业的「房间里的大象」

企业每年在培训上投入巨额预算，但培训效果评估始终是一道未解的难题。

```
传统培训评估的困境：

    讲师在台上激情澎湃              学员在群里热火朝天
         ↓                              ↓
    "大家有问题吗？"                "收到" "好的" "+1"
         ↓                              ↓
    全场沉默 ×                      表面繁荣 ×
         ↓                              ↓
    ┌─────────────────────────────────────────┐
    │  结营时才发现：                           │
    │  · 谁真正投入了？——不知道                │
    │  · 学习效果如何？——靠感觉                │
    │  · 如何量化提升？——Excel 手动统计        │
    │  · 怎么激励学员？——发红包，效果递减      │
    └─────────────────────────────────────────┘
```

**核心痛点一览：**

| 痛点 | 现状 | 后果 |
|:---:|:---:|:---:|
| 评估滞后 | 结营后才统计 | 错过干预窗口 |
| 维度单一 | 只看签到和考试 | 忽视过程表现 |
| 人工成本高 | 助教全程记录 | 疲惫且遗漏 |
| 学员无感 | 不知道自己表现如何 | 缺乏参与动力 |
| 数据孤岛 | 散落在各平台 | 无法沉淀复用 |

> *我们需要的不是另一个打卡工具，而是一个真正理解「学习行为」的智能系统。*

## 三、核心功能亮点

### 1. 全域感知：AI 无声记录一切

系统像一位不知疲倦的「超级助教」，7x24 小时在群里默默工作。

每一条消息、每一张图片、每一个文件、每一次互动——**全部自动捕获，零遗漏。**

学员感知到的是正常的群聊氛围，背后却有一套精密的行为分析引擎在运转。不打扰，不侵入，不改变任何人的使用习惯。

```
学员发了一张 PPT 截图
    ↓
系统自动识别：这是一份「学习成果展示」
    ↓
归入「成果展示」维度，+8 分
    ↓
排行榜实时更新 ← 整个过程 < 3 秒
```

---

### 2. 五维评分模型：看见完整的学习画像

我们不相信单一分数能衡量学习效果。系统围绕 **5 大能力维度** 展开评估，每个维度包含多个细分评分项，总计 **15 项精细指标**：

```
                        ★ 思维深度 ★
                       ╱            ╲
                      ╱              ╲
              ★ 协作能力 ★          ★ 执行力 ★
                      ╲              ╱
                       ╲            ╱
                ★ 影响力 ★ ———— ★ 创新力 ★
```

| 维度 | 代号 | 评分项（共 15 项） | 说明 |
|:---:|:---:|:---|:---|
| **知识力** | K | K1 每日签到、K2 随堂测验、K3 知识总结、K4 AI 纠错 | 学了多少、理解了多少 |
| **实操力** | H | H1 作业提交、H2 实操截图、H3 视频打卡 | 有没有动手做、交作业 |
| **创新力** | C | C1 创意分享、C2 点赞互动、C3 深度创作 | 有没有创意和新想法 |
| **社交力** | S | S1 群消息互动、S2 互评贡献 | 活不活跃、有没有帮助别人 |
| **成长力** | G | G1 视频学习、G2 课外资源分享、G3 持续活跃 | 有没有反思进步、分享资源 |

每位学员的成长轨迹都会被绘制成 **雷达图**，一目了然地呈现优势与短板。

### 3. AI 多模态分类：不只是关键词匹配

这不是一个简单的「关键词计数器」。

系统采用 **规则引擎 + LLM 双引擎架构**：简单场景走规则引擎（快速、零成本），复杂内容交给 AI 大模型（GLM-4 Vision）进行多模态理解。

**AI 能理解什么？**

- 一张 PPT 截图 → 识别为「成果展示」而非普通聊天图片
- 一段代码截图 → 识别为「工具实操」并归入执行力维度
- 一段长文思考 → 区分「深度发言」和「日常闲聊」
- 一个表情回复 → 区分「点赞认可」和「无意义水群」

```
┌──────────────────────────────────────────────────┐
│                消息分类流水线                       │
│                                                  │
│  输入消息 ──→ [规则引擎] ──→ 匹配成功？           │
│                              │                   │
│                         是 ──┘── 否              │
│                         ↓        ↓               │
│                    直接评分    [AI 分类引擎]        │
│                              GLM-4 Vision        │
│                                  ↓               │
│                            返回分类结果           │
│                                  ↓               │
│                            评分 + 入库            │
└──────────────────────────────────────────────────┘
```

### 4. 赛博朋克实时大屏：让数据燃起来

评分不应该藏在后台报表里。我们设计了一面 **赛博朋克风格的实时 Dashboard**——投射到培训教室的大屏上，让每一次积分变动都成为全场焦点。

**大屏核心模块：**

- **实时排行榜** —— 按段位分组展示，分数变化自带动效，名次变动一目了然
- **个人详情页** —— 五维 AQ 雷达图 + 维度得分分解 + 成长时间线
- **段位晋升动画** —— 升级瞬间全屏播放赛博朋克风格的晋升特效
- **勋章墙** —— 根据行为自动授予成就徽章，可视化成长足迹

**视觉风格关键词：** 霓虹辉光 / 深色背景 / 数据粒子流 / 科幻 HUD 界面

> *想象一下：学员刚在群里发了一段精彩分析，3 秒后大屏上他的名字闪着霓虹光向上跳动——这就是即时正反馈的力量。*

### 6. 零代码管理：培训师的超级武器

培训师不需要懂技术。所有管理操作都通过两种方式完成：

**方式一：群聊关键词指令**

```
管理员在群里发送关键词，系统即时响应：

  管理 / 管理面板 / 控制面板   → 管理员控制面板（开期、开窗口、毕业结算）
  测验 / 随堂测验 / 考试      → 从飞书多维表格题库抽题，发送互动答题卡片
  互评 / 互评投票 / 投票      → 学员互评投票卡片，自动计算社交力得分
  看板 / 排行 / 排行榜        → 战绩天梯榜卡片，一键跳转 Web Dashboard
  调分 / 手动调分             → 运营手动调分卡片（可绕过自动评分上限）
  成员 / 成员管理             → 成员角色管理卡片（隐藏排行/更改角色）
```

**方式二：飞书多维表格管理题库**

### 7. 防刷机制：公平是游戏化的基石

游戏化最怕的就是「刷分」。系统内置了多重防护：

- **每期上限** —— 每个评分项每期有独立分数上限（`perPeriodCap`），防止刷分
- **幂等去重** —— 同一消息不会重复计分（`sourceRef` 唯一性约束）
- **AI 质量判定** —— 6 项需 LLM 审核的指标经过 AI 内容质量评估
- **管理员审核队列** —— AI 评分结果推送审核卡片，一键批准/驳回
- **运营手动调分** —— 管理员可绕过自动上限进行手动加减分修正

```
│  IM 集成     │  飞书开放平台 SDK                              │

├─────────────┼───────────────────────────────────────────────┤

│  部署方式    │  systemd 进程管理 / Nginx 反向代理 / Let's Encrypt │

└─────────────┴───────────────────────────────────────────────┘
```

**为什么选择这套技术栈？**

- **TypeScript 全栈** —— 前后端统一语言，降低维护成本
- **SQLite** —— 零运维，单文件即数据库，培训场景完美适配
- **Fastify** —— 性能是 Express 的 2 倍，插件生态成熟
- **飞书 SDK** —— 原生集成，无需额外开发 IM 通道
- **GLM-4V** —— 国产多模态大模型，合规、低成本、高精度

## 八、如何开始使用

### 快速启动（3 步完成）

```
步骤 1：创建飞书机器人
────────────────────
在飞书开放平台创建应用，获取 App ID 和 App Secret
开启机器人能力，配置消息接收权限

步骤 2：部署系统
────────────────────
git clone <repo-url>
cp .env.example .env          # 填入飞书凭证和 AI API Key
npm install && npm run build
systemctl start ai-seed-project    # systemd 管理进程

步骤 3：开始使用
────────────────────
将机器人拉入培训群
在飞书多维表格中导入题库
打开 https://orz.md/dashboard 查看实时大屏
```
