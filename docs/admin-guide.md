> 适用对象：训练营讲师、运营人员（无需任何技术背景）
> 
> 所有操作均在飞书群聊中完成
> 
> 文档版本：2026-04-13（已根据代码实际实现校准）

## 一、系统概览

### 这个系统能做什么？

我们的飞书 Bot 是一个**自动评分助手**。它住在飞书群「AI 训练营」里，默默地做这些事：

- **自动捕获**学员在群里发的每一条消息、图片、文件、表情反应
- **自动识别**消息类型（签到、作业、创意分享、互动等）
- **自动打分**或交给 AI 评分后推送审核卡片给管理员
- **自动更新**排行榜和段位

**简单来说：学员在群里正常学习互动，系统自动记录和打分，管理员只需要偶尔看看审核卡片、点点按钮。**

### 五大评分维度

系统从五个角度评价学员的表现：

| 代号    | 维度名 | 一句话解释          |
|:-----:|:---:|:-------------- |
| **K** | 知识力 | 学了多少、理解了多少     |
| **H** | 实操力 | 有没有动手做、交作业     |
| **C** | 创新力 | 有没有创意和新想法      |
| **S** | 社交力 | 群里活不活跃、有没有帮助别人 |
| **G** | 成长力 | 有没有反思进步、分享资源   |

### 五个段位等级

学员的综合表现越好，段位越高：

```
🌱 AI潜力股  →  🔬 AI研究员  →  🎯 AI操盘手  →  🧠 AI智慧顾问  →  ⚡ AI奇点玩家
   (起步)         (入门)          (熟练)          (创造性应用)       (顶尖水平)
```

### AQ 综合指数

AQ 是学员的「综合能力得分」，由五个维度的分数加权计算得出。AQ 越高，排名越靠前，段位晋升越快。你可以把 AQ 理解为学员的「AI 能力值」。

## 二、日常操作指南

### 2.1 发送管理面板（已实现）

**什么时候用：** 需要开启新周期、开启评分窗口、查看系统状态、触发毕业结算时。

**怎么操作：**

1. 打开飞书群「AI 训练营」
2. 在群里直接发送以下任一关键词：`管理`、`管理面板`、`控制面板`
3. Bot 会立刻回复一张管理面板卡片

```
┌───────────────────────────────────┐
│         ⚙️ 管理员面板              │
│                                    │
│  📊 当前系统状态                    │
│  活跃周期：第 3 期                  │
│  活跃窗口：W2（进行中）             │
│  学员人数：12/14                    │
│  待审核事件：3 条                   │
│                                    │
│  📅 开启新周期                      │
│  选择周期：[ ▼ ]   [🟢 开启此周期]  │
│                                    │
│  📦 开启评分窗口                    │
│  选择窗口：[ ▼ ]   [🟢 开启此窗口]  │
│                                    │
│  🎓 毕业结算                        │
│  [🔴 触发毕业结算]                  │
│                                    │
│  [🔄 刷新状态]                      │
└───────────────────────────────────┘
```

在管理面板上你可以：

- **开启新周期** — 每次课结束后，选择下一个周期编号（1-12），点「开启此周期」。系统会自动关闭上一周期。
- **开启评分窗口** — 选择窗口编号（W1-W5 或 FINAL），点「开启此窗口」。每个窗口包含 2 个周期。
- **触发毕业结算** — 所有课程结束后（Period 12 之后），点此完成最终段位评定。此操作不可撤销。
- **刷新状态** — 随时查看系统当前状态

> 注意：只有「运营」和「讲师」身份的人发关键词才会弹出面板，学员发这些词不会触发。

### 2.2 发起随堂测验（已实现）

**什么时候用：** 课堂上想做一次小测验时。

**怎么操作：**

1. 在群里直接发送以下任一关键词：`测验`、`随堂测验`、`考试`
2. Bot 会自动从题库中抽取**当前周期**的测验题
3. 群里弹出测验卡片，学员点击选项作答
4. 答对自动得分（K2 知识维度），无需管理员手动操作

> 如果发了「测验」但没有弹出卡片，可能是当前周期还没有录入题目，请参考下方「测验题管理」章节。
> 
> 权限要求：仅「运营」和「讲师」可发起。

### 2.3 发起互评投票（已实现）

**什么时候用：** 想让学员互相评价、投票选出优秀同学时。

**怎么操作：**

1. 在群里直接发送以下任一关键词：`互评`、`互评投票`、`投票`
2. Bot 弹出投票卡片，列出所有学员名单
3. 学员选择自己认为表现优秀的同学（最多选 3 人，若学员不足 3 人则按实际人数）
4. 投票结束后系统自动计算结果

> 权限要求：仅「运营」和「讲师」可发起。

### 2.6 审核 AI 评分（已实现）

部分复杂内容（如知识总结、创意分享等）会由 AI 先打分，然后推一张审核卡片到群里。

```
┌─────────────────────────────────┐
│  待审核：张三 提交了 K3 知识总结    │
│  AI 评分：3 分                     │
│  AI 评语：内容详实，符合要求         │
│                                    │
│     [ ✅ 批准 ]    [ ❌ 驳回 ]      │
└─────────────────────────────────┘
```

**操作：** 看一眼 AI 评语，觉得合理就点「批准」，不合理就点「驳回」。

> AI 判断通常比较准确，大部分情况快速浏览后点批准即可。

### 2.7 手动调分（已实现）

**什么时候用：** 发现分数有误需要修正，或想给学员额外加分/扣分时。

**怎么操作：**

1. 在群里发送以下任一关键词：`调分`、`手动调分`
2. Bot 弹出调分卡片，包含三个下拉菜单：
   - **选择成员** — 从学员列表中选择
   - **选择评分项** — K1~G3 共 15 项
   - **分值变化** — 从预设值选择（+1/+2/+3/+5/+10/-1/-2/-3/-5）
3. **依次选择**三个下拉菜单后，点击「✅ 确认调分」
4. 系统显示 toast 确认消息

> ⚠️ 重要：必须**先选下拉菜单，再点按钮**。下拉选择的值会临时缓存 10 分钟，过期需重新选择。
> 
> 权限要求：仅「运营」和「讲师」可使用。

### 2.8 成员管理（已实现）

**什么时候用：** 需要修改某人的角色，或从排行榜隐藏/显示某人时。

**怎么操作：**

1. 在群里发送以下任一关键词：`成员`、`成员管理`
2. Bot 弹出成员管理卡片，上方显示所有成员列表，下方有两个下拉菜单：
   - **选择成员** — 从列表中选择要操作的人
   - **选择操作** — 隐藏/取消隐藏/改为学员/改为运营/改为讲师/改为旁听
3. **依次选择**两个下拉菜单后，点击「✅ 确认操作」
4. 系统显示 toast 确认消息

**支持的操作：**

| 操作          | 说明                     |
| ----------- | ---------------------- |
| 隐藏（从排行榜移除）  | 该成员不再出现在排行榜上，但数据保留     |
| 取消隐藏（恢复排行榜） | 重新在排行榜上显示该成员           |
| 改为学员        | 将角色设为学员（会被评分）          |
| 改为运营        | 将角色设为运营（不会被评分，可使用管理功能） |
| 改为讲师        | 将角色设为讲师（不会被评分，可使用管理功能） |
| 改为旁听        | 将角色设为旁听（不会被评分）         |

> ⚠️ 重要：必须**先选下拉菜单，再点按钮**。与调分卡片一样，下拉选择值缓存 10 分钟。
> 
> 权限要求：仅「运营」和「讲师」可使用。

## 三、测验题管理

### 题目存在哪里？

测验题目存储在**飞书多维表格**（也叫 Base / Bitable）中。系统通过环境变量 `FEISHU_BASE_QUIZ_TABLE` 指定题库表格的位置。

**不需要写任何代码，直接编辑表格即可。**

> 如果发「测验」后 Bot 回复「题库未配置」，说明环境变量 `FEISHU_BASE_QUIZ_TABLE` 尚未设置，请联系技术支持。

### 表格字段说明

| 列名（字段）   | 填什么               | 示例                |
|:-------- |:----------------- |:----------------- |
| **期数**   | 填数字，表示第几期课的题目     | `3`               |
| **题目**   | 题目的完整文字           | `以下哪个不是生成式AI的应用？` |
| **选项A**  | 第一个选项的文字          | `文本生成`            |
| **选项B**  | 第二个选项的文字          | `图像识别`            |
| **选项C**  | 第三个选项的文字          | `代码编写`            |
| **选项D**  | 第四个选项的文字          | `数据分析`            |
| **正确答案** | 填 A、B、C 或 D（大写字母） | `B`               |

### 如何添加新题目

1. 打开飞书，找到题库所在的多维表格
2. 在表格末尾新增一行
3. 按照上面的字段说明，逐列填写内容
4. 保存（自动保存）
5. 下次在群里发「测验」时，Bot 会**实时读取**该期数的题目（无缓存，改了立刻生效）

### 如何修改已有题目

直接在表格中找到要修改的行，修改对应单元格即可，改完自动生效。

### 题库工作原理

Bot 收到「测验」关键词后，会根据**当前活跃周期编号**去飞书多维表格中查询「期数」字段匹配的所有记录，将它们组装成测验卡片发到群里。因此：

- 题目是**实时从表格读取**的，不存在缓存延迟
- 如果当前周期是第 3 期，Bot 就只会读取「期数 = 3」的行
- 如果找不到匹配的行，Bot 会回复「第 X 期暂无测验题目」

### 注意事项

- **期数必须填对**：Bot 是根据当前活跃周期来读取题目的。如果第 3 期的课要考试，期数就填 `3`
- **一期可以有多道题**：同一个期数可以添加多行，每行是一道题
- **选项至少填两个**：A 和 B 必填，C 和 D 可以留空（但推荐四个都填）
- **正确答案只填字母**：填 `A`、`B`、`C` 或 `D`，不要填选项内容

## 四、评分规则说明

### 15 个评分项一览

系统共有 15 个评分项，分布在 5 个维度。每个评分项有固定的基础分和每期上限。

#### K 维度 — 知识力

| 代号  | 评分项       | 基础分 | 每期上限 | 触发方式                                         |
|:---:|:--------- |:---:|:----:|:-------------------------------------------- |
| K1  | 每日签到      | +3  | 3    | 自动 — 学员在群里发任何消息即签到                           |
| K2  | 随堂测验      | +10 | 10   | 自动 — 学员完成测验后根据答题情况计分                         |
| K3  | 知识总结      | +3  | 3    | AI评分 — 学员发 50 字以上文字（无特殊关键词时默认归类），AI 评估后推审核卡片 |
| K4  | AI纠错/深度回答 | +4  | 4    | AI评分 — 学员发含「纠错」「错误」「修正」「改正」关键词的 20 字以上文本     |

#### H 维度 — 实操力

| 代号  | 评分项     | 基础分 | 每期上限 | 触发方式                                        |
|:---:|:------- |:---:|:----:|:------------------------------------------- |
| H1  | 作业/文件提交 | +5  | 5    | 自动 — 学员在群里发送文件附件                            |
| H2  | 实操截图/作品 | +3  | 3    | AI评分 — 学员发送图片消息，AI 评估后推审核卡片                 |
| H3  | 视频打卡    | +2  | 2    | 自动 — 学员发含「完成视频」「视频完成」「看完了」「学完了」「视频打卡」关键词的消息 |

#### C 维度 — 创新力

| 代号  | 评分项          | 基础分 | 每期上限 | 触发方式                                      |
|:---:|:------------ |:---:|:----:|:----------------------------------------- |
| C1  | 创意分享         | +4  | 8    | AI评分 — 学员发含「创意」「创新」「新玩法」「妙用」关键词的 20 字以上文本 |
| C2  | 点赞/表情互动      | +1  | 4    | 自动 — 学员给他人消息添加表情反应                        |
| C3  | 深度创作(Prompt) | +5  | 5    | AI评分 — 学员发含「prompt」「提示词」「模板」关键词的 20 字以上文本 |

#### S 维度 — 社交力

| 代号  | 评分项     | 基础分 | 每期上限 | 触发方式                    |
|:---:|:------- |:---:|:----:|:----------------------- |
| S1  | 群消息互动   | +3  | 6    | 自动 — 通过卡片互动（签到、作业提交等）触发 |
| S2  | 帮助他人/互评 | +2  | 2    | 自动 — 参与互评投票时触发          |

#### G 维度 — 成长力

| 代号  | 评分项    | 基础分 | 每期上限 | 触发方式                                           |
|:---:|:------ |:---:|:----:|:---------------------------------------------- |
| G1  | 视频学习完成 | +5  | 5    | 自动 — 学员发含「完成」+「视频/学习」组合关键词的消息（与 H3 互斥，H3 优先匹配） |
| G2  | 课外资源分享 | +3  | 6    | AI评分 — 学员分享 URL 链接，AI 评估后推审核卡片                 |
| G3  | 持续活跃   | +4  | 4    | 自动 — 系统根据学员跨期活跃度自动计算                           |

### 哪些是全自动的？哪些需要管理员审核？

**全自动（9 个）** — 学员操作后立刻得分，无需管理员操作：

> K1、K2、H1、H3、C2、S1、S2、G1、G3

**AI 评分 + 管理员审核（6 个）** — AI 先打分，推审核卡片，管理员点「批准」或「驳回」：

> K3、K4、H2、C1、C3、G2

### Bot 是怎么自动识别消息类型的？

消息分类器按以下优先级依次判定（一条消息可以同时触发多个评分项）：

1. **视频关键词** — 含「完成视频/视频完成/看完了/学完了/视频打卡」→ H3 视频打卡；含「完成」+「视频/学习」→ G1 视频学习
2. **URL 链接** — 包含 `http://` 或 `https://` → G2 课外资源分享
3. **文件附件** — 消息类型为 file 或含非图片附件 → H1 作业提交
4. **图片消息** — 消息类型为 image → H2 实操截图
5. **长文本（20字以上）** — 根据关键词判断优先级：含「prompt/提示词/模板」→ C3，含「纠错/错误/修正/改正」→ K4，含「创意/创新/新玩法/妙用」→ C1，50字以上无特殊关键词 → K3 知识总结
6. **任何消息** — 都算一次签到（K1），每期上限 3 分

> 同一条消息可以同时触发多个评分项（如发一张图片会同时触发 H2 和 K1），但 Bot 只回复一条确认消息（优先显示高价值评分项）。

### 防刷机制

- 每个评分项对每个学员有**每期上限**。达到上限后继续提交不计分。
- 同一条消息（同一 sourceRef）不会重复计分（幂等性保护）。
- Bot 的自动回复有 5 分钟频率限制：同一学员同一评分项在 5 分钟内只回复一次确认。

## 五、常见问题（FAQ）

### Q：学员发了消息，但好像没有被记录怎么办？

**可能原因：**

- 学员可能还未注册。新学员在群里发第一条消息时会自动注册，请让学员再发一条消息试试。
- 消息可能已被记录但未达到显示阈值。打开排行榜查看该学员是否已有分数。
- 当前周期可能未开启。在管理面板中确认是否有活跃周期。
- 若当前为破冰期（第 1 期），所有提交保留但不计入 AQ。

---

### Q：如何添加新学员？

不需要手动添加。**新学员在群里发第一条消息时会自动注册**，系统会自动从飞书获取其姓名和头像。

---

### Q：发了「测验」但没有弹出测验卡片？

请检查以下几点：

1. **是否有权限**：只有「运营」和「讲师」身份才能发起测验
2. **题库是否配置**：如果 Bot 回复「题库未配置」，说明环境变量 `FEISHU_BASE_QUIZ_TABLE` 未设置，请联系技术支持
3. **题库是否有题目**：打开飞书多维表格，确认当前周期编号下有录入题目
4. **期数是否匹配**：题目的「期数」字段要和当前活跃周期一致。比如现在是 Period 3，题目的期数也要填 `3`

---

### Q：如何手动调整学员分数？

在群里发 `调分`，Bot 会弹出调分卡片。依次选择成员、评分项、分值变化，然后点确认即可。

---

### Q：排行榜上学员显示 0 分？

学员可能还没有在群里发送过任何消息。鼓励学员先在群里发一条消息完成签到。

---

### Q：审核卡片太多，能不能批量通过？

目前需要逐条审核。好消息是 AI 判断通常比较准确，大部分情况下快速点批准就行。

---

### Q：排行榜打不开了？

## 六、注意事项

### 重要提醒

1. **Bot 只在群聊中工作**
   Bot 不会处理私聊消息。所有评分和互动必须在飞书群「AI 训练营」中进行。

2. **讲师和运营人员不会被评分**
   系统会根据角色自动过滤。讲师和运营人员在群里发的消息不会触发评分流程。

3. **关键词是触发词，请注意使用场景**
   以下词语会触发系统功能，在日常聊天中请避免单独使用：
   
   | 关键词                          | 功能    | 权限要求  |
   | ---------------------------- | ----- | ----- |
   | `管理` / `管理面板` / `控制面板`       | 管理员面板 | 运营/讲师 |
   | `测验` / `随堂测验` / `考试`         | 随堂测验  | 运营/讲师 |
   | `互评` / `互评投票` / `投票`         | 互评投票  | 运营/讲师 |
   | `看板` / `排行` / `排行榜` / `成长看板` | 战绩天梯榜 | 所有人   |
   | `调分` / `手动调分`                | 手动调分  | 运营/讲师 |
   | `成员` / `成员管理`                | 成员管理  | 运营/讲师 |
   
   > 系统只在消息**恰好等于**这些关键词时才触发（@机器人 前缀会被自动去除）。如果你在正常讨论中需要用到这些词，请放在句子中间使用（如「我们来讨论一下管理方法」），而不是单独发一个「管理」。

4. **每个评分项有每期上限**
   学员不会因为频繁操作而无限加分。每个评分项在每个周期内都有分数上限。

### 2.9 AI 助教 Chat Bot（已实现）

**什么时候用：** 学员或管理员有任何问题想问 AI 时。

**怎么操作：**

在飞书群里 **@ 机器人** 并跟上你的问题即可，例如：

> @辉瑞 HBU AI 训练营评估系统 什么是 RAG？

Bot 会以**奇点小助**的身份回答，并鼓励群内其他同学补充。

**使用规则：**

- 必须显式 @ 机器人才会响应，直接发问题不会触发
- 对学员：Bot 会引导思考，不直接给作业答案/测验答案
- 对管理员（运营/讲师）：Bot 可以给更直接的专业回答
- Bot 有短期记忆：**5 分钟内**的追问会保持上下文（最多记住 3 轮对话）
- 限流：同一个人 **30 秒冷却 + 每小时 20 次**；全群每分钟最多 30 次

**@Bot 消息不会触发评分**：避免"@Bot 问问题"也算一次签到的双重计费。

**启用/关闭：** 通过环境变量 `FEISHU_CHAT_BOT_ENABLED=true/false` 控制。需要同时配置 `FEISHU_BOT_OPEN_ID`（Bot 自身的飞书 open_id）。
